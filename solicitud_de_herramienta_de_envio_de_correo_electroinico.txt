Herramienta de correo electrónico para mercadotecnia directa.
Objetivo de la solicitud.
Contar con una plataforma que permita planificar, diseñar, ejecutar y medir campañas de correo electrónico de forma centralizada, segura y eficiente.
Antecedentes (situación actual).
Actualmente la Subdirección de Comunicación opera las campañas a través de un contrato de mercadotecnia directa (CT-0407-24), para el canal de correo el proveedor cubre las necesidades que van desde generación de plantillas html, repositorio de imágenes y plantillas, gestor de plantillas, validación de bases de datos, carga de bases de datos, monitoreo de los envíos, reportes y gestión de incidentes.


Alcance.


Herramienta de servicio que deberá:
* Permitir incluir elementos interactivos dentro del correo electrónico, de manera enunciativa mas no limitativa, encuestas, sondeos, puntos de opinión, actualización de datos, así como mensajes personalizados y contenido dinámico.[a]
* Contemplar la administración y operación, mediante acceso WEB.
 
Este servicio deberá incluir las funcionalidades de administración, operación y monitoreo de envío de correos, diseño y elaboración de las plantillas para correo electrónico y servicio de soporte técnico.


Funcionalidades.
Plataforma Web de administración de campañas de correo electrónico la cual deberá permitir el acceso mediante un usuario y contraseña.

	La plataforma debe de permitir la carga de archivos pdf que se adjuntarán en campañas. Los archivos podrán ser personalizados, es decir; uno por cada registro (correo electrónico). El peso máximo será de 10 MB, dichos archivos adjuntos podrán contener información dinámica.[d][e]
	Se validará la carga de archivos en formato pdf.

	Carga exitosa de base de datos en distintos formatos (txt, xls o xlsm)[f]
	Además de permitir subir la base de datos directamente se podrá realizar la integración con el sistema de datos de contacto del Instituto (DACI)
	Conexión con DACI
	Se deberá contar con un módulo, aplicación o servicio validador preventivo de bases de contactos de correo que permita identificar las cuentas que son susceptibles de recibir mensajes. La herramienta deberá verificar (sin enviar un correo electrónico) la sintaxis de la cuenta, que el dominio exista, comunicación SMTP para averiguar si el servicio está disponible, cuentas duplicadas, buzones llenos, dominios de Catch-All (ejemplo: yopmail), correos trampa (spamtrap). Asi mismo deberá de validar y evitar la duplicidad de correos electrónicos en las listas.[g][h]
	Se cargará una base de datos con registros erróneos. 
	Se deberá de tener la gestión de usuarios fatals con la finalidad de asegurar que no se realicen reintentos a estos correos.[i][j]
	Se validará que usuarios que solicitaron no recibir nuestras campañas no se les envíe correos.
	Integración con diseños que permitan crear correos electrónicos adaptables a diferentes dispositivos (pantallas) como lo son: diseños responsivos y adaptables; así como diferentes visualizaciones: modo oscuro, modo claro, vista vertical y vista horizontal. Considerar elementos como la legibilidad del texto, la claridad de las imágenes, la facilidad de navegación y la optimización para dispositivos móviles.[k][l]
	Se realizarán pruebas con diferentes plantillas correspondientes a las campañas recurrentes del instituto como promoción de eventos, estado de cuenta, productos de solución o de crédito.
	Repositorio para imágenes de las plantillas de correo las cuales se podrán visualizar desde la red interna y externa.[m][n]

	El sistema debe permitir programar el envío en fechas y horas específicas.[o][p]
	Validación de la programación de una campaña.

	Se verificará que el porcentaje de envio se esté actualizando cuando se envíen las campañas.
	Los reportes de la herramienta deberán permitir un análisis de las campañas a través de los siguientes indicadores por campaña: registros cargados, enviados, aperturas totales, aperturas únicas, clics totales, clics únicos, rebotes, correos no válidos, bajas por spam, anulación de suscripción.
	Se enviarán campañas de pruebas y se consultarán reportes.
	Los reportes detallados deberán contar con la siguiente información por registro (correo electrónico): estatus del envío (registro no valido, rebotado, enviado, apertura, baja por spam o anulación de la suscripción), clics totales, clics únicos (identificar sobre que enlace dio clic el usuario), fecha y hora de envío, fecha y hora de entrega, fecha y hora de apertura, fecha y hora de clic. El reporte se podrá exportar en formato xls.
	Se enviarán campañas de pruebas y se descargarán reportes.


	Realizar actividades de monitoreo y mantenimiento para asegurar que las direcciones IP, desde las cuales se envían los correos electrónicos, mantengan buena reputación y no se incluyan en las listas de los servicios Black List disponibles a nivel nacional e internacional, y en caso de que ocurra, se deberá corregir la situación con la finalidad de asegurar la continuidad del servicio.
	Este punto solo se podrá valida una vez que inicie la operación, al enviar campañas productivas.
	Se deberán monitorear los envíos para asegurar su entrega y en caso de que una campaña sea pausada, realizar las actividades necesarias para que esta sea concluida.
	Este punto solo se podrá valida una vez que inicie la operación, al enviar campañas productivas.
